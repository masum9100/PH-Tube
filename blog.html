1. Discuss the scope of var, let, and const

JavaScript provides three ways to declare variables: var, let, and const. These keywords have different scopes and behaviors.

- var: Function-scoped, accessible throughout the entire function.

- let: Block-scoped, limited to the block where it's defined.

- const: Block-scoped, like let, but with values that cannot be reassigned.

2. Use Cases of null and undefined

In JavaScript, both null and undefined are used to represent the absence of a value, but they have different use cases:

- null: It's a deliberate absence of any object value. You can assign it to a variable to indicate that it has no value.

- undefined: It typically represents a variable that has been declared but not initialized or a non-existent property of an object.

3. What is a REST API?

REST (Representational State Transfer) is an architectural style for designing networked applications. A REST API is a set of rules and conventions for building and interacting with web services.

- RESTful APIs use standard HTTP methods (GET, POST, PUT, DELETE) to perform CRUD (Create, Read, Update, Delete) operations on resources.

- They are stateless, meaning each request from a client to a server must contain all the information needed to understand and process the request.

- Resources in a REST API are identified by URLs, and data is typically exchanged in JSON format.
